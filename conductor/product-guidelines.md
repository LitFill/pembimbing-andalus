# Product Guidelines: Pembimbing Andalus

## Tone and Style
- **Voice**: Clean, professional, and helpful.
- **Style**: Use concise language. Avoid jargon where possible.
- **Clarity**: Prioritize information density while maintaining readability.

## User Experience (UX) Principles
- **Mobile First**: Design every interaction with a touch interface in mind.
- **Instant Feedback**: Provide visual cues for all user actions, such as copying data or expanding accordion sections.
- **Zero-Latency Search**: The search and filter functionality must respond instantly as the user types.
- **Predictability**: Keep the layout consistent across different view modes (Table vs. Card).

## Visual Guidelines
- **Minimalist Aesthetic**: Focus on content over decorative elements.
- **High Contrast**: Ensure text is easily readable on all backgrounds, following basic accessibility standards.
- **Visual Consistency**: Use a unified color palette and spacing system throughout the application.

## Accessibility (a11y)
- **Semantic HTML**: Use proper tags (e.g., `<table>`, `<button>`, `<h2>`) to ensure compatibility with screen readers.
- **Keyboard Navigation**: Ensure all interactive elements are reachable and usable via keyboard.
- **ARIA Labels**: Use `aria-label` and other ARIA attributes where necessary to provide additional context.
